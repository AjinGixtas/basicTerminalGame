[color=cyan]ls[/color] [color=yellow][directory][/color] – List all files and directories in the current directory.

[color=cyan]cd[/color] [color=yellow][directory][/color] – Change to the specified directory.

[color=cyan]pwd[/color] – Print the current working directory.

[color=cyan]echo[/color] [color=yellow][message][/color] [color=yellow][-n][/color] [color=yellow][-e][/color]
	[color=green]Outputs[/color] the specified [color=yellow][message][/color] to the terminal.
	[color=yellow]Arguments:[/color]
	  [color=yellow]-n[/color] – Optional. Suppress the trailing newline (e.g., `echo -n "Hello"` prints `Hello` without a newline).
	  [color=yellow]-e[/color] – Optional. Enable escape sequences like `\n` (newline), `\t` (tab), etc.

[color=cyan]clear[/color] – Clear the terminal screen.

[color=cyan]mkdir[/color] [color=yellow][directory_name][/color] – Create a new directory with the specified name.

[color=cyan]rmdir[/color] [color=yellow][directory_name][/color] – Remove an empty directory.

[color=cyan]scan[/color] [color=yellow][-d [depth_val]][/color] [color=yellow][-v][/color]
	[color=green]Scans[/color] for vulnerabilities or network connections.
	[color=yellow]Arguments:[/color]
	  [color=yellow]-d [depth_val][/color] – Optional. Set the scan depth (number of levels to scan).
	  [color=yellow]-v[/color] – Optional. Enable verbose output (shows detailed information).

[color=cyan]crack[/color] [color=yellow][init][/color] [color=yellow][hostname_or_ip][/color] OR [color=yellow][flags [lock_value]][/color]
	[color=green]Crack[/color] different types of security locks on a specified system or service.
	[color=yellow]Usage:[/color]
	  [color=yellow]crack init [hostname_or_ip][/color] – Begin the cracking process by specifying a hostname or IP address. This initializes the cracking attempt for the target system.
	  [color=yellow]crack -[lock_flag] [lock_value][/color] – Specify the type of lock (e.g., [color=yellow]i5[/color], [color=yellow]p9[/color]) and provide the value to attempt cracking the lock.
	[color=green]Example:[/color]
	  [color=yellow]crack init 192.168.1.1[/color] – This initializes the cracking attempt on the system at `192.168.1.1`.
	  [color=yellow]crack -i5 3 -p9 7[/color] – This attempts to crack an [color=yellow]i5[/color] lock with value [color=yellow]3[/color] and a [color=yellow]p9[/color] lock with value [color=yellow]7[/color] on the same system.

[color=cyan]connect[/color] [color=yellow][ip_or_hostname][/color] – Connect to a remote host. Use a neighboring [color=yellow]hostname[/color], or any [color=yellow]IP address[/color].
	[color=green]Example:[/color]
	  [color=yellow]connect 192.168.1.1[/color] – Connect to the host at IP address `192.168.1.1`.
	  [color=yellow]connect neighborhost[/color] – Connect to a neighboring host directly using its hostname.


[color=cyan]analyze[/color] [color=yellow][ip_or_hostname][/color]
	[color=green]Analyzes[/color] the target host or the current system for information such as its security features.
	[color=yellow]Arguments:[/color]
	  [color=yellow][ip_or_hostname][/color] – Optional. The target IP address or neighboring hostname. If omitted, the current system will be analyzed.
	[color=green]Output:[/color]
	  Displays information about the system, such as:
		- IP address
		- Hostname
		- Host type (e.g., [color=purple]PERSON[/color], [color=purple]BUSINESS[/color])
		- Defense value
		- Security type
